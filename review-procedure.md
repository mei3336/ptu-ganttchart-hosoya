# Skill: レビュー手順（エンジニア初心者向け）

数学の証明を読むときのように「前提 → 処理 → 結論」の順で確認する。

## レビューの手順

### 1. 前提を確認する
- コメントに書かれている「この関数が何を保証するか」を読む
- 関数名・変数名から想像した動きと、コメントの内容が一致しているか

### 2. 処理の流れを確認する
- 1関数1責務になっているか（複数の役割を持っていないか）
- 変数名が明確で、似たような名前が使われていないか（CLAUDE.mdの命名規約）
- マジックナンバーがないか

### 3. 結論（動作結果）を確認する
- AIが報告した「動作確認の方法」を実際に自分でも再現する
  （例：DevToolsを開いてIndexedDBの中身を見る）
- 「何ができるようになったか」のサマリーと、実際の動きが一致しているか

### 4. わからない箇所があったら
- 「なぜこう書いたのか」をAIにその場で聞き直す（コメントだけで判断できないときは遠慮せず質問する）
- 一度で全部理解しようとせず、工程ごと（Skills単位）に区切って確認する

## NGサイン（要注意）
- コメントがない、または「〇〇する処理」のような説明になっていないコメント
- 1つの関数が長く、何をしているか一読で分からない
- 似た名前の変数が複数あり、区別がつかない
