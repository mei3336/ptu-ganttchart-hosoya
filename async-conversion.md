# Skill: 非同期化（コールバック → Promise/async-await）

## いつ使うか
コールバック形式のコードが出てきた、または生成されたコードがコールバック地獄になっているとき

## AIへの依頼テンプレート
```
以下のコールバック形式のコードを、Promiseでラップしてasync/awaitで
書き直して。書き直した理由（なぜasync/awaitの方が読みやすいか）も
コメントで一言添えて。

【対象コード】
（コードを貼る）
```

## 確認すること（レビューポイント）
- 書き直し前と書き直し後で「動作が変わっていない」と説明されているか
- エラー時の処理（reject / catch）が元のコードから漏れていないか

## 次のステップ
書き直しが完了したら、元の呼び出し元コードも動作確認する
