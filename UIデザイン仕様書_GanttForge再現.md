# UIデザイン仕様書：GanttForge再現アプリ

## 0. 本書の位置づけ・使い方

- 本書は、`ganttforge.html`の**見た目（配色・余白・寸法・フォント）を数値として抜き出した仕様書**である
- **Claude Codeへの指示には、`ganttforge.html`のソースコードそのものを渡さない。本書に書かれた「値」と、基本設計書の画面構成（2章）・詳細設計書の各機能仕様のみを渡す**
- ソースコードは、実装後の見た目を見比べる「答え合わせ」用として、Mei自身の手元にのみ置く
- 理由：ソースをそのまま渡すと「写経」になり、設計書を自分の言葉で組み立てる力（本研修の主目的）が育たないため。CLAUDE.mdの命名規約・コメント方針と同様、UI面でも「値を渡して、実装はAIに考えさせる」形を徹底する

## 2. レイアウト構造（基本設計書2.1・2.2の再掲＋寸法補足）

### 2.1 全体構成（上から順）

```
┌─────────────────────────────────────────────┐
│ 第1ヘッダー（グローバル操作・高さ目安 上記hdr-h相当） │
│  左：ロゴ／プロジェクト選択／＋新規プロジェクト        │
│  右：データ取込／データ出力／PDF出力／Excel出力        │
├─────────────────────────────────────────────┤
│ 第2ヘッダー（プロジェクト操作バー・高さ34px）          │
│  左：ロック／＋スケジュール追加／表示粒度／今日／…      │
│  右：スナップショット／即時メモ／メモ／…              │
├───────────────┬─────────────────────────────┤
│ WBSパネル       │ ガントチャート本体                │
│ （幅520px固定）  │ ・ヘッダー高さ82px（年月＋日付）    │
│ ・スケジュール名  │ ・1行の高さ44px                  │
│ ・開始/終了/状態  │ ・バー、マイルストーンマーカー等    │
└───────────────┴─────────────────────────────┘
```

### 2.2 サイドパネル類（コメント・変更履歴・メモ等）

- 画面右側からスライドインする形式のパネル（`tp-panel`相当のクラス構造）
- 幅は機能ごとに異なる（目安：タスク詳細720px、コメント一覧700px、メモ560px、即時メモ440px、スナップショット460px）
- パネル左端に「掴んで幅を変更できるハンドル」を持つ（ドラッグでリサイズ可能、最小280px〜画面幅の90%まで）

### 2.3 モーダル・ポップアップ

- 画面中央に表示、背景に半透明のオーバーレイ
- モーダル外側（オーバーレイ部分）をクリックすると閉じる
- 影は「大」を使用し、他要素より明確に浮き上がって見えるようにする

### 2.4 トースト通知

- 画面右下に固定表示
- 表示時は下からスライドイン、約2.8秒で自動的にスライドアウトして消える
- 背景は濃い文字色（`#111827`相当）、文字は白

## 3. ガントチャート特有の視覚仕様

| 要素 | 仕様 |
|---|---|
| 親行のバー | 高さがやや高め・角丸大きめ、行の中央に配置。バー内に「スケジュール名 (子タスク合計日数)」を表示 |
| 子（葉）行のバー | 親行より薄い高さで、行の65%あたりの位置に配置。バー幅が60px超の場合のみ「名前 (日数)」ラベルを表示 |
| バーの左右端 | ホバー時のみうっすら暗く色付き、掴んでドラッグすると開始日／終了日を個別に変更できる |
| 完了ステータスのバー | 不透明度を下げて表示（未着手・進行中との視覚的な差別化） |
| 土日・祝日の列 | タイムライン背景に薄い色（土＝薄青、日＝薄赤、祝＝薄橙）を敷く |
| マイルストーンマーカー | ひし形アイコン＋縦の点線ガイド。ホバーで日付ラベルのツールチップ表示 |
| ロック中の表示 | ヘッダーとガントの間に薄赤帯＋赤ボーダー、「🔒ロック中—編集操作は無効です」の文言 |

## 4. Claude Codeへの指示方針（実装依頼時の進め方）

1. **本書（デザイン仕様書）と基本設計書2章（画面構成）・データモデル設計書・詳細設計書を渡す。`ganttforge.html`のコードは渡さない**
2. 依頼テンプレート例：
   ```
   以下の仕様に沿って、GanttForgeのガントチャート画面のHTML/CSSを実装して。
   ・配色・寸法は「UIデザイン仕様書」の値をそのまま使う
   ・レイアウト構造は「基本設計書2章」の通り
   ・ライブラリはCDN経由のみ、ビルドツールは使わない（CLAUDE.md準拠）
   ・見た目の細かいピクセル単位の再現よりも、配色・階層構造・余白のリズムが仕様書と一致していることを優先して
   ```
3. 実装結果をブラウザで見た上で、Mei自身が`ganttforge.html`と目視比較し、差分があれば「〇〇の色が違う」「余白が狭い」等、**具体的な指摘として**Claude Codeにフィードバックする（ソースコードを見せて直させるのではなく、差分を言葉で伝える）
4. どうしても言葉で伝えづらい微妙なズレ（影の濃さ、フォントの太さ等）に限り、該当箇所のスクリーンショットを見比べ材料として使うのは可（コードそのものではなく画像で伝える）

## 5. 本書に含めなかった情報（意図的にスコープ外）

- アイコンの具体的なSVGパス（アイコンは「同じ意味が伝わる別のアイコン」で代替可とし、ピクセル単位の完全一致は求めない）
- CSSの詳細なセレクタ構造・クラス名（実装はClaude Codeに任せ、命名はCLAUDE.mdの命名規約に従わせる）
- アニメーションの正確なイージング関数の値（「なめらかに」「スッと」等の言葉での指示に留める）

これらは「値をコピーする」のではなく「意図を伝えて実装させる」という本書の方針上、あえて数値までは詰めていない。
